# 贡献指南
感谢您能够为仓库做出贡献，您可以在本仓库中分享您的语法文件。

在本指南，您将会了解到：
- [前置工作](#前置工作)
- [语法文件命名](#语法文件命名)
- [文件更改](#文件更改)
- [代码规范](#代码规范)

## 前置工作
您需要将 MT 管理器升级至 2.16.0 及其以上版本，本仓库所需要的语法文件均为 2.16.0 以上版本提供。

您还需要学习写一个 .mtsx 语法文件，访问MT管理器官方的 [用户手册-语法文件开发](https://mt2.cn/guide/file/mt-syntax.html) 了解如何开发语法文件。

## 语法文件命名
为了统一语法文件的命名，语法文件以 `后缀名.mtsx` 作为文件名。

## 文件更改
1. 你需要 fork 本仓库，随后在您的 fork 仓库中添加/更改语法文件
2. 提交更改，git commit 描述内容为 `更改/新增语法高亮`
3. 点击 Contribute 按钮发布到 Pull Request 并等待代码审核

## 代码规范
为了更好维护语法文件，您需要遵守以下的代码规范：

1. 遵守统一的语法文件命名
2. 统一缩进，建议使用二缩进，不得使用 Tab 缩进 + 空格缩进混用
3. 语法文件存放在仓库的 mtsx 文件夹中
4. 对于每一个正则表达式的匹配板块，需要添加对应的注释
5. 避免因为政策表达式而导致的性能浪费
6. 文件头需要遵守以下格式
   ```
   // require MT >= 2.16.0
   // Syntax: 语法名称
   // Version: 版本号
   // License: 协议
   // Repo: https://github.com/wisebreeze/MT-syntax
   ```

## 帮助

如果您贡献的内容无另外声明，将遵循本仓库的 MPL 2.0 协议
